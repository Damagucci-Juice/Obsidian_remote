# TDD + 행동 = BDD

- 행동 주도 개발(BDD)
- 테스트 코드 시나리오 단위로 작성
- 시나리오는 BDD, 테스트는 TDD

# BDD를 적용해보며

- TC로 고민하는 시간은 비용
- BDD는 기획서가 TS, 고민하는 시간이 줄어든다.
- 테스트 코드의 의도가 명확해 진다.
- 기획을 이해하며 테스트 코드 작성 가능, 탄탄한 설계의 시작
- 기능 추가 리팩토링시 버그 감소
- 팀을 위해서
